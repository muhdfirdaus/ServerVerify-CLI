## Project Overview
**ServerVerify-CLI** is a lightweight, zero-dependency Python auditing tool designed for automating hardware validation in an x86 server manufacturing environment (e.g., within a Supermicro, Dell, or HPE integration context). 

During manufacturing run-in or final quality control (QC), it is critical that the physical hardware being tested strictly matches the expected Bill of Materials (BOM) without relying on manual and error-prone BIOS screen inspections. This tool performs an immediate cross-check of core components (CPU core count, physical Memory capacity, and Disk volume detection) directly from the OS level. It dumps actionable results for downstream data collection or MES integration.

## Key Features
* **Reduces False Failures:** Replaces volatile bash pipelines (like `awk`/`grep` chains) that often break across different Linux distributions or OS updates. Employs direct subprocess captures and resilient Python parsing.
* **Accelerates Test Cycle Time:** Sub-second execution speeds up the diagnostic stage on the floor. Removes the need for bloated third-party agent installations.
* **Zero Dependencies:** Written entirely in standard Python. No need to manage virtual environments or a `requirements.txt` on factory floor staging servers.
* **Auditable Logs & Machine-Readable Output:** Generates a human-readable `test_results.log` for floor operators troubleshooting live issues, while simultaneously writing `hardware_audit.json` for telemetry ingestion.
* **Security Conscious:** Operates strictly on argument lists (`shell=False`) to prevent shell injection vectors common in older test automation scripts.

## Installation
Since ServerVerify-CLI relies exclusively on Python's Standard Library alongside common Linux utility binaries (`lscpu`, `free`, `lsblk`), no complicated setup or package manager intervention is required.

```bash
# Clone the repository to the test unit or staging server
git clone https://github.com/your-org/ServerVerify-CLI.git
cd ServerVerify-CLI

# Ensure the script is executable
chmod +x hardware_audit.py
```

## Usage

ServerVerify-CLI is designed to be executed out-of-the-box on the device under test (DUT). Simply run the script as the designated test user.

```bash
# Run the audit script directly
python3 hardware_audit.py
```

## Integrating into Production (Senior Perspective)

To maximize the value of this validation script at scale, it should not just remain a standalone operator tool. Here is how it can fit into the larger manufacturing pipeline:

### MES (Manufacturing Execution System) Integration
In a mature factory environment, the `hardware_audit.json` payload generated by this script can be automatically pushed to a central MES REST API via `curl` or native Python requests. 
* **Benefit:** This eliminates operator data-entry errors and permanently binds the hardware validation result directly to the server's serial number (SN), chassis ID, or work order before the unit is packaged. If the payload indicates `"status": "FAIL"`, the MES can automatically mark the asset as non-shippable and route it to debugging.

### PXE Boot / Live OS Environments
For bare-metal deployment and high-throughput validation without installing a persistent OS:
1. The server boots into a lightweight ephemeral Linux image via the network (PXE / iPXE).
2. A generic `systemd` test service dynamically executes `hardware_audit.py` on startup. 
3. Variables (`EXPECTED_CPU_CORES`, `EXPECTED_RAM_MB`) could be dynamically injected into the script over the network based on the unit's MAC address profile.
4. Pass/Fail results are communicated to the build engineer via IPMI/BMC alerts (e.g., turning the chassis UID LED red) or immediately halting the line by powering down the node upon failure.
